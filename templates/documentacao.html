Visão Geral

O Pushdown Operator é uma proposta de extensão de expressões regulares através de operadores especiais que permitem denotar linguagens livres de contexto (context-free languages). Esta ferramenta didática e de pesquisa implementa os conceitos propostos no trabalho de conclusão de curso de Guilherme da Cruz Oliveira, apresentado à Universidade Federal do Tocantins em 2024.

Diferentemente de expressões regulares tradicionais, que reconhecem apenas linguagens regulares, o Pushdown Operator permite definir linguagens livres de contexto através de uma notação compacta e intuitiva, gerando automaticamente os autômatos com pilha (Pushdown Automata - PDA) correspondentes.

Objetivos Principais

  • Fornecer uma notação concisa para especificar linguagens livres de contexto
  • Gerar automaticamente autômatos com pilha a partir de expressões
  • Facilitar o ensino de linguagens formais e teoria da computação
  • Apoiar pesquisa em propriedades de linguagens livres de contexto
  • Permitir análise e visualização de autômatos gerados

Definições Formais

Linguagens Livres de Contexto

Uma linguagem é livre de contexto se existe uma gramática livre de contexto que a gera, ou equivalentemente, se existe um autômato com pilha que a reconhece. Formalmente:

Definição: Uma linguagem L é livre de contexto se existe um autômato com pilha (PDA) M tal que L = L(M), onde L(M) é a linguagem aceita por M.

Autômato com Pilha (PDA)

Um autômato com pilha é uma máquina abstrata que estende autômatos finitos com uma pilha (stack) para armazenar símbolos.

Definição: Um autômato com pilha é uma tupla M = (Q, Σ, Γ, δ, q₀, Z₀, F) onde:
  • Q é um conjunto finito de estados
  • Σ é o alfabeto de entrada
  • Γ é o alfabeto da pilha
  • δ: Q × (Σ ∪ {ε}) × Γ → P(Q × Γ*) é a função de transição
  • q₀ ∈ Q é o estado inicial
  • Z₀ ∈ Γ é o símbolo inicial da pilha
  • F ⊆ Q é o conjunto de estados finais

Transição de Pilha

Uma transição em um PDA é representada como:

δ(q, a, X) = {(q', α₁), (q'', α₂), ...}

Onde:

  • q é o estado atual
  • a ∈ Σ ∪ {ε} é o símbolo lido (ou ε para transição vazia)
  • X ∈ Γ é o símbolo desempilhado
  • q' é o novo estado
  • α é a sequência de símbolos a empilhar

Operadores e Sintaxe

O Pushdown Operator propõe uma extensão de expressões regulares com operadores especiais para manipular a pilha:

Operadores Básicos

Operador Símbolo Descrição Formal Exemplo Precedência
União + L₁ ∪ L₂ a+b 0 (menor)
Concatenação . L₁ · L₂ a.b 1
Fechamento de Kleene * L* a* 2
Sharp (Pilha) # Operador de pilha para balanceamento a#b → {aⁿbⁿ | n ≥ 0} 5 (maior)
Two-Points (Pilha) : Operador de pilha com símbolo BOS (ab):a#b 3
Separador ; Alternativas de operações de pilha a:(b#c;d#e) 4

Caracteres Especiais

  • & — Epsilon (ε), representa a cadeia vazia
  • \ — Escape para caracteres literais (ex.: \), \+, \#)
  • ( ) — Parênteses para agrupamento e precedência

Definição Formal dos Operadores de Pilha

Sharp (#): Dados dois PDAs M₁ e M₂, o operador sharp M₁#M₂ cria um novo PDA que reconhece cadeias onde a sequência de símbolos empilhados por M₁ é desempilhada por M₂, garantindo balanceamento.
Two-Points (:): O operador two-points M₁:M₂ combina M₁ com M₂, introduzindo o símbolo BOS (Bottom-Of-Stack) 'Z' para garantir aceitação por pilha vazia ao final da computação.

Exemplos de Expressões

Expressões Formais
# Linguagem {aⁿbⁿ | n ≥ 0}
(&:a#b)

# Linguagem {aⁿbⁿ | n ≥ 1}
(ab):a#b

# Linguagem {a²ⁿbⁿ | n ≥ 0}
&:(aa)#b

# Palíndromos sobre {a,b}
(&+a+b):(a#a;b#b)

# Linguagem {aⁿbⁿcⁿ | n ≥ 0}
((&:a#b):c#c)

# Expressão aritmética com parênteses balanceados
(x(\+x)*):((x\+)*\()#(\)(\+x)*)(\+(...)*)*

Arquitetura do Sistema

Pipeline de Processamento

O sistema implementa um pipeline de compilação que transforma expressões em autômatos com pilha:

Expressão Infixa (entrada do usuário)
↓
explicitarConcatenacao() — Insere operadores de concatenação implícitos
↓
infixaToPosfixa() — Converte para notação pós-fixa (Shunting-yard customizado)
↓
executarPosfixa() — Executa a pós-fixa construindo o PDA
↓
Automato (saída — PDA completo)
↓
imprimir_imagem() — Gera visualização via Graphviz
prepararXML() / salvarXML() — Exporta para formato JFLAP

Componentes Principais

expression.py

Implementa o compilador de expressões. Responsável pela conversão de expressões em PDAs:

  • explicitarConcatenacao(infixa) — Insere operadores de concatenação implícitos
  • infixaToPosfixa(infixa) — Converte infixa para pós-fixa (Shunting-yard customizado)
  • executarPosfixa(posfixa) — Executa a pós-fixa construindo o PDA
  • convertExpressionInPA(expression, path=None) — Interface de alto nível
gerador_de_automatos.py

Implementa os operadores que constroem PDAs compostos:

  • base(a) — PDA básico para um símbolo
  • concatenacao(a1, a2) — Concatenação de PDAs
  • uniao(a1, a2) — União de PDAs
  • fechoDeKleene(a1) — Fechamento de Kleene
  • sharp(a1, a2) — Operador sharp para balanceamento
  • two_points(a1, a2) — Operador two-points com BOS
Automato.py

Define a classe que representa um PDA:

  • alfabeto — Σ (alfabeto de entrada)
  • estados — Q (conjunto de estados)
  • transicoes — δ (função de transição)
  • estadoInicial — q₀ (estado inicial)
  • estadosFinais — F (estados finais)
  • auxiliar — Γ (alfabeto da pilha)

Métodos importantes:

  • imprimir() — Exibe detalhes no console
  • imprimir_imagem(name) — Gera PNG via Graphviz
  • prepararXML() — Retorna string XML JFLAP
  • salvarXML(saida, path) — Salva XML em disco
  • executar(palavra) — Testa se uma palavra é aceita
pdogex.py

API de alto nível com interface similar ao módulo re do Python:

  • compile(expression) — Compila uma expressão
  • fullmatch(string) — Teste de aceitação completa
  • find(string) — Busca primeira ocorrência
  • find_all(string) — Busca todas as ocorrências
  • replace(string, replace) — Substitui primeira ocorrência
  • replace_all(string, replace) — Substitui todas as ocorrências
  • split(string) — Divide string por padrão
  • subn(string, replace) — Substitui e retorna contagem
Transicoes.py

Classe que representa uma transição do autômato com pilha:

  • Estado de origem
  • Símbolo de entrada
  • Símbolo a desempilhar
  • Estado de destino
  • Símbolo a empilhar
Pilha.py

Implementação de uma pilha (stack) usada internamente para processamento de expressões.

app.py

Aplicação Flask que fornece interface web para o sistema:

  • GET/POST / — Página principal para compilar expressões
  • GET/POST /testar — Página para testar cadeias contra o autômato

API de Programação

Compilação de Expressões

from pdogex import pdogex

p = pdogex()
p.compile("(&:a#b)")  # Compila expressão para {aⁿbⁿ | n ≥ 0}

Teste de Aceitação

# Teste completo (fullmatch)
print(p.fullmatch(""))         # True (n=0)
print(p.fullmatch("ab"))       # True (n=1)
print(p.fullmatch("aabb"))     # True (n=2)
print(p.fullmatch("aab"))      # False (desbalanceado)

Busca e Correspondência

# match() retorna posição do primeiro match
pos = p.match("xxaabbxx")  # 2

# search() retorna tupla (start, end)
pos = p.search("xxaabbxx")  # (2, 6)

# search_all() retorna lista de tuplas
positions = p.search_all("aabbaaabbb")  # [(0, 4), (4, 8)]

# finditer() retorna dicionários com span e match
for item in p.finditer("aabbaaabbb"):
    print(item)  # {'span': (0, 4), 'match': 'aabb'}

# Encontra primeira ocorrência
pos = p.find("xxaabbxx")       # (2, 6)

# Encontra todas as ocorrências
matches = p.find_all("aabbaaabbb")  # ["aabb", "aaa", "bbb"]

Substituição

# Substitui primeira ocorrência
resultado = p.replace("xxaabbxx", "MATCH")  # "xxMATCHxx"

# Substitui todas as ocorrências
resultado = p.replace_all("aabbaaabbb", "X")  # "XX"

# subn() retorna (resultado, contagem)
resultado, count = p.subn("aabbaaabbb", "X")  # ("XX", 2)

Divisão

# split() divide string por padrão
partes = p.split("aabbxxaabbxx")  # ["", "xx", "xx"]

Memoização

A classe pdogex usa memoização automática para acelerar buscas repetidas:

p = pdogex()
p.compile("(&:a#b)")

# Primeira chamada: executa o autômato
p.fullmatch("aabb")  # Lento

# Segunda chamada: usa cache
p.fullmatch("aabb")  # Rápido

# Limpar cache se necessário
p.memo.clear()

Exemplos Práticos

Exemplo 1: Linguagem {aⁿbⁿ | n ≥ 0}

Reconhecer cadeias balanceadas de "a" e "b":

from pdogex import pdogex

p = pdogex()
p.compile("(&:a#b)")

print(p.fullmatch(""))         # True
print(p.fullmatch("ab"))       # True
print(p.fullmatch("aabb"))     # True
print(p.fullmatch("aaabbb"))   # True
print(p.fullmatch("aab"))      # False

Exemplo 2: Palíndromos

Reconhecer palíndromos sobre {a, b}:

p = pdogex()
p.compile("(&+a+b):(a#a;b#b)")

print(p.fullmatch(""))         # True
print(p.fullmatch("a"))        # True
print(p.fullmatch("aba"))      # True
print(p.fullmatch("abba"))     # True
print(p.fullmatch("aabaa"))    # True

Exemplo 3: Linguagem {a²ⁿbⁿ | n ≥ 0}

Reconhecer cadeias com "a" em quantidade par (2n) e "b" em quantidade n:

p = pdogex()
p.compile("&:(aa)#b")

print(p.fullmatch(""))         # True (n=0)
print(p.fullmatch("aab"))      # True (n=1, 2 a's)
print(p.fullmatch("aaaabb"))   # True (n=2, 4 a's)
print(p.fullmatch("aaaaaabbb")) # True (n=3, 6 a's)
print(p.fullmatch("aab"))      # False (1 a, não 2n)

Exemplo 4: Expressões Aritméticas

Reconhecer expressões aritméticas com parênteses balanceados:

p = pdogex()
# Expressão: x(+x)* com parênteses balanceados
p.compile("(x(\+x)*):((x\+)*\()#(\)(\+x)*)(\+(...)*)*")

# Testes
print(p.fullmatch("x"))           # True
print(p.fullmatch("x+x"))         # True
print(p.fullmatch("(x+x)"))       # True
print(p.fullmatch("((x+x)+x)"))   # True
print(p.fullmatch("(x+x"))        # False (parêntese não fechado)

Exemplo 5: Processamento de Texto

Encontrar e substituir padrões em um texto:

p = pdogex()
p.compile("a:(b#c)")

texto = "meu nome é bac bac bac"

# Encontrar todas as ocorrências
matches = p.find_all(texto)
print(matches)  # ['bac', 'bac', 'bac']

# Substituir todas
resultado = p.replace_all(texto, "XYZ")
print(resultado)  # "meu nome é XYZ XYZ XYZ"

# Contar substituições
resultado, count = p.subn(texto, "XYZ")
print(f"Substituídas {count} ocorrências")

FAQ & Resolução de Problemas

Problema: O binário dot do Graphviz não está instalado ou não está no PATH.

Solução: Instale Graphviz no seu sistema operacional (veja seção de Instalação acima).

Verificar instalação:

which dot    # Linux/macOS
where dot    # Windows

Problema: A expressão tem parênteses desbalanceados.

Solução: Verifique se todos os parênteses abertos têm fechamento correspondente.

Exemplos incorretos:

  • (a+b — Falta fechar parêntese
  • a+b) — Parêntese sem abertura
  • ((a+b) — Parêntese extra aberto

Exemplos corretos:

  • (a+b)
  • ((a+b)+c)
  • (a+(b+c))

Problema: Preciso usar caracteres como +, #, * como símbolos literais.

Solução: Use barra invertida \ para escapar:

\+    # Símbolo + literal
\#    # Símbolo # literal
\*    # Símbolo * literal
\(    # Parêntese literal
\)    # Parêntese literal

Exemplo: Expressão aritmética com operador +

x(\+x)*  # Reconhece: x, x+x, x+x+x, ...

# — Operador de pilha básico. Empilha símbolos da primeira expressão e desempilha com a segunda.

a#b  # Reconhece a^n b^n (n ≥ 0)

: — Operador two-points. Combina com o símbolo BOS (Bottom-Of-Stack) 'Z' para garantir aceitação por pilha vazia.

(ab):a#b  # Reconhece a^n b^n (n ≥ 1) com prefixo "ab"

; — Separador para múltiplas operações de pilha alternativas.

a:(b#c;d#e)  # Reconhece "bac" ou "dae"

& representa epsilon (ε), a cadeia vazia. É usado para:

  • Transições sem ler símbolo de entrada
  • Operações de pilha sem empilhar/desempilhar
  • Permitir zero repetições em operadores
(&:a#b)   # Aceita cadeia vazia (n=0)
&*        # Aceita apenas cadeia vazia
a&b       # Equivalente a "ab" (& é ignorado)

Automaticamente: Ao compilar uma expressão, a imagem é gerada automaticamente em static/automato_output.png.

from expression import convertExpressionInPA

posfixa, automato = convertExpressionInPA("(&:a#b)")
# Imagem gerada em: static/automato_output.png

Manualmente: Você pode gerar a imagem em qualquer momento:

automato.imprimir_imagem("meu_automato")
# Gera: static/meu_automato.png

Automaticamente: Especifique um caminho ao compilar:

from expression import convertExpressionInPA

posfixa, automato = convertExpressionInPA("(&:a#b)", "meu_automato.jflap")
# Arquivo salvo em: static/meu_automato.jflap

Manualmente:

xml = automato.prepararXML()
automato.salvarXML(xml, "./static/meu_automato.jflap")

Abra o arquivo em JFLAP para visualizar e editar o autômato.

Complexidade: As operações de busca (find_all, replace_all) têm complexidade O(n²) no tamanho da string, pois testam todas as substrings possíveis.

Otimizações:

  • Memoização automática de resultados
  • Cache de substrings já testadas
  • Limpar cache se necessário: p.memo.clear()

Recomendação: Para strings muito grandes (> 10.000 caracteres), considere dividir em chunks menores.
